features

Everything's encrypted before the server sees it. Same vault for everything; no separate password manager + notes app + file drop.

Notes
Markdown editor with live preview. [[wiki-links]] with autocomplete and broken-link rendering. Backlinks panel. Daily notes, templates.
Tasks
Checkboxes, due dates with overdue / today / future colour cues, one-click toggle from the list, sorted open-first.
Lists
Drag-reorder checklists for shopping, packing, anything. Enter to add, Backspace to remove, "clear N done" sweep.
Secrets
Passwords + TOTP codes (live with countdown). Generator with charset toggles. Copy with 30s clipboard clear, reveal with 10s auto-hide. HIBP k-anonymity breach check (the password never leaves the device).
Files
Drag & drop upload, encrypted on the client, content-addressed blobs on disk. Inline preview for image / PDF / text. 50 MB cap.
Snippets & Bookmarks
Code blocks with language tag, copy button. URL bookmarks with notes. Same encrypted body as everything else.
Tags + path tree + search
Sidebar with type / tag / path filters. Cmd-K palette searches across all items by title, dispatches actions, opens shares, scans backlinks.
Version history
Every body save snapshots the previous ciphertext. Restore any version; the restore is itself versioned. No silent overwrite.
Share via link
Notes and files. Random share key in the URL fragment — the server never sees it. Optional expiry. Recipient needs no account.
Team spaces
Per-item key sealed once per current member with libsodium crypto_box_seal. Atomic invite re-wrap. Owner / editor / viewer roles. Move items between spaces.
Auth
Argon2id-derived KEK from passphrase. Recovery code as second escrow. Multiple WebAuthn passkeys with PRF-derived KEK — no per-passkey master key duplication. Auto-lock after 15 min idle.
CLI companion
rongnote login / ls / cat / new / spaces / use. Same E2E crypto as the browser. Single 4 MB binary. Useful for cron, scripted ingest, headless servers.
Audit log
Every auth event and every secret read is recorded. Each user sees their own log; never anyone else's. Plain notes / files / tasks reads are not recorded — too noisy and a tempting honeypot.
Encrypted export
One-click tar bundle of everything. The bytes are the same ciphertext the server holds — your master key on a device decrypts the bundle.
Theme + font
Light / dark / auto, persisted. Font size step adjustable from the status bar. No third-party fonts, just the OS monospace stack.
Mobile responsive
Stack mode below 700px: list or detail, never both. Slide-in sidebar drawer. Touch-friendly drag-reorder. Searches via a synthetic Cmd-K dispatched from a button.

what the server can never see

The server stores ciphertext, key wraps, and metadata you opted into being plaintext (titles, tags, paths, timestamps, file sizes). Everything else lives behind libsodium primitives the server can't run.

passphrase ──Argon2id──► passphrase_kek
recovery code ──Argon2id──► recovery_kek
                              │
master_key (random, 32 bytes) ─┘
   │
   ├─ secretbox-wrap by passphrase_kek   → master_wrap_passphrase
   ├─ secretbox-wrap by recovery_kek     → master_wrap_recovery
   ├─ keyed BLAKE2b("rongnote-auth-v1")  → auth_hash (server-side Argon2id of this)
   └─ wraps each per-item item_key:
         personal space → secretbox(item_key, master_key)
         team space     → crypto_box_seal(item_key, member.public_key) per member

item bodies: secretbox(plaintext, item_key)
files:       secretbox(file_bytes, item_key)  — content-addressed by sha256 of ciphertext
shares:      secretbox(payload, share_key)    — share_key lives in the URL fragment

A leaked database without a passphrase or recovery code is opaque. A leaked share token without the URL fragment is opaque. A removed team member sees no future updates. SECURITY.md has the full threat model.

self-host in three commands

# 1. one compose file
curl -O https://raw.githubusercontent.com/operator64/rongnote/main/docker-compose.example.yml
mv docker-compose.example.yml docker-compose.yml

# 2. one secret
echo "NOTES_DB_PW=$(openssl rand -base64 24)" > .env

# 3. up
docker compose up -d
# → http://localhost:8080

The image bundles the SvelteKit SPA inside the Rust binary via rust-embed; one container serves both. Postgres in a sidecar. deploy.md covers Cloudflare tunnel + Traefik, blob backups, and reverse-proxy notes.

Built and pushed by GitHub Actions on every push to main. Multi-arch image at ghcr.io/operator64/rongnote-server:latest.

auth
passphrase, recovery code, multi-passkey via WebAuthn PRF
cli
separate rongnote-cli workspace member, RustCrypto stack
